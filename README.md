# SC2002 HDB Management System (CLI)

This repository contains my **SC2002 Java OOP project**, developed as part of NTU coursework. The project is a command-line interface (CLI) system that simulates an HDB housing application workflow with multiple user roles.

## Project Overview

The system models a simplified HDB application process where different types of users interact with the platform based on their roles. It demonstrates object-oriented programming principles such as encapsulation, inheritance, and role-based access control.

There are three main user roles:

- **Applicant** – Applies for flats, views application status, and manages personal requests  
- **Officer** – Assists in processing applications and handles booking-related tasks  
- **Manager** – Oversees the system, manages applications, and handles approvals  

## Features

- Role-based login system (Applicant / Officer / Manager)
- CLI-based navigation and interaction
- HDB flat application workflow simulation
- Application submission, viewing, and status tracking
- Officer assignment and processing logic
- Manager-level approval and oversight functions

## Concepts Used

This project was built to practice core OOP concepts in Java:

- Classes and objects  
- Inheritance and polymorphism  
- Encapsulation and access control  
- Separation of concerns  
- Basic system design using layered logic  

## Tech Stack

- Java  
- Object-Oriented Programming (OOP)  
- Command Line Interface (CLI)
